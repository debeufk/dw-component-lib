# Contributing

If you are looking to contribute to this package than this page contains all the needed information.

## 1. Prerequisites
 - Node.js (recomended version v12 LTS)
 - Yarn (recomended version v1.22.15)

## 2. Minimal path to awesome
Use the steps below to install and run the code.
 1. Pull the code from [GitHub](https://github.com/dlw-digitalworkplace/dw-component-lib)
 2. Run `yarn install` in the root folder of the project
 3. Run dd`yarn build` in the root of the folder
 4. Run `yarn start` in the root of the folder.
 5. Choose an application you want to start
	1. @dlw-digitalworkplace/dw-component-lib - starts the storybook
	2. @dw-react-app/demo-spfx-app - runs a demo SPFx project which uses the component library

> **Warning**: Errors might occur when using `yarn install` in PowerShell. Best to use CMD for installation purposes.

### 1. Branching
The default branch of this project is `master`.
This branch always contains the latest production code.

If you want to contribute to this repo, you should create a new branch that starts from `master`.
Depending on the the type of contribution the naming convention is as follows:

| Branch naming                | Used for                                           															    |
|------------------------------|--------------------------------------------------------------------------------------|
| features/name-of-the-feature |  <ul><li>New components</li><li>New packages</li><li>New demo's</li><li>...</li></ul>|
| bugs/name-of-the-bug         |  <ul><li>Component bugfixes</li><li>Documentation fixes</li><li>...</li></ul>				|

### 2. Creating a PR
Follow the steps below to submit your contribution to the project

#### Prepare the PR
1. Make sure all code is commited and pushed to your topic branch
2. Create the change file: run `yarn change`
3. Select the correct change type
4. Describe the change that is made
5. Push the commit that is generated by the change command

#### Create the PR
1. Navigate to the PR section on GitHub
2. Create a new PR from your topic branch into master
3. Fill in the PR template (be as descriptive as possible) and creat the PR
4. Feel good about yourself because you made a contribution
